06 — Motion & Spacing
Motion Principles
FieldMarq is a professional tool. Motion must be purposeful — it communicates state change, not personality. Every animation should make the platform feel faster, not more animated.
Interaction Value Purpose
Hover states 0.15s ease Color changes, opacity shifts
Hover lift translateY(-1px) Cards and primary buttons
Modal entrance scale(0.97→1) + opacity, 0.2s Modals, drawers, popovers
Skeleton loader shimmer 1.5s infinite Loading states only
✓ Do
·0.15s ease on all transitions
·Reduced motion media query
·Motion that communicates meaning
✗ Don't
·Bounce or spring easings
·Animations over 0.3s on UI
·Motion on every interaction

07 — What FieldMarq Is Not
Anti-Patterns
These patterns are the statistical center of AI-generated design. Each one signals that no designer was involved. Avoid unconditionally.
✗ Purple gradient on white
The signature of every AI-built SaaS tool in 2024–25. Instantly marks FieldMarq as undifferentiated.
✗ Inter / Roboto / Arial
The default fonts of AI-generated interfaces. Using them is opting into mediocrity by default.
FieldMarq
✗ Three-column icon card grid
Every SaaS landing page. Every AI dashboard. No differentiation. No editorial voice.
⚡
📊
🎯
✗ Full-app dark mode
Content areas with tables and reports are harder to read on dark backgrounds. Dark mode is shell-only.
Table data is harder to read here →
✗ Pill-shaped primary buttons
Reserve pill shapes for status badges. Primary CTAs use 8px radius — confident, not casual.
✗ Heavy drop shadows
Dated. Use whitespace and 1px borders as separation. Shadows should be invisible — felt, not seen.

08 — Brand Shape
The FieldMarq Shape
One shape. Eight variants. The wedge is always the left edge — ombre from aqua to deep teal. Top-right corner is always rounded. All other corners are hard right angles. Card body never overlaps the wedge. Scale width and height freely. Gradient stops and wedge angle never change.
With Header Bar — Light & Dark
Use when the box needs a label, section title, or category context.
Without Header Bar — Thick Wedge
Use when no label is needed. Content leads directly.
Without Header Bar — Thin Wedge
Most minimal. Use for inline callouts, data sheet annotations, supporting copy.
Developer Reference — CSS
Shared geometry — all variants
/* Wedge taper — clip-path used on ::after pseudo-element
   Top-right rounded via border-radius on the card
   Card body path never overlaps wedge — starts at wedge right edge

   THICK WEDGE: 26px wide  clip-path: polygon(0 0,100% 0,100% 84%,0 100%)
   THIN WEDGE:  12px wide  clip-path: polygon(0 0,100% 0,100% 90%,0 100%)

   GRADIENT STOPS — never change:
   #00E5C4 0% · #00C9A7 30% · #0097A7 65% · #006073 88% · #00353F 100%

   TYPE SCALE inside all variants:
   Title:   Cabinet Grotesk 900 · 20px · -0.8px tracking · #0D1F24 / white
   Body:    Satoshi 400 · 13px · #4A6670 / rgba(255,255,255,0.45)
   CTA:     Satoshi 700 · 12px · #0097A7 / #00C9A7
   Divider: 1px rule above CTA · #E2EAEC / rgba(255,255,255,0.08)
   Padding: 24px all sides inside card body */
✓ Do
·Top-right rounded — always
·All other corners hard right angles
·Card body starts at wedge right edge
·24px internal padding minimum
✗ Don't
·Fill the bottom-left corner behind the wedge
·Change wedge angle or gradient stops
·Use on every element — scarcity is the point
·Mix thick and thin wedge in the same view
