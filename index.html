06 — Motion & Spacing
Motion Principles
FieldMarq is a professional tool. Motion must be purposeful — it communicates state change, not personality. Every animation should make the platform feel faster, not more animated.
Interaction Value Purpose
Hover states 0.15s ease Color changes, opacity shifts
Hover lift translateY(-1px) Cards and primary buttons
Modal entrance scale(0.97→1) + opacity, 0.2s Modals, drawers, popovers
Skeleton loader shimmer 1.5s infinite Loading states only
✓ Do
·0.15s ease on all transitions
·Reduced motion media query
·Motion that communicates meaning
✗ Don't
·Bounce or spring easings
·Animations over 0.3s on UI
·Motion on every interaction

07 — What FieldMarq Is Not
Anti-Patterns
These patterns are the statistical center of AI-generated design. Each one signals that no designer was involved. Avoid unconditionally.
✗ Purple gradient on white
The signature of every AI-built SaaS tool in 2024–25. Instantly marks FieldMarq as undifferentiated.
✗ Inter / Roboto / Arial
The default fonts of AI-generated interfaces. Using them is opting into mediocrity by default.
FieldMarq
✗ Three-column icon card grid
Every SaaS landing page. Every AI dashboard. No differentiation. No editorial voice.
⚡
📊
🎯
✗ Full-app dark mode
Content areas with tables and reports are harder to read on dark backgrounds. Dark mode is shell-only.
Table data is harder to read here →
✗ Pill-shaped primary buttons
Reserve pill shapes for status badges. Primary CTAs use 8px radius — confident, not casual.
✗ Heavy drop shadows
Dated. Use whitespace and 1px borders as separation. Shadows should be invisible — felt, not seen.
